JavaScript（一）

一、起源

起源于前端验证（无需请求服务器）

三个部分构成： ECMAScript、DOM、BOM

特点：解释性语言（不用编译） 动态语言 基于原型的面向对象

二、JS编写位置

  1. 可以将js代码写到标签内
  2. 可以写到js标签内
  3. 可以写在外部文件，然后通过script标签引入

    写到外部文件中可以在不同页面中同时引用

三、基本语法

  1. JS中严格区分大小写

  2. JS中每一条语句以分号（；）结尾 （如果不写分号，浏览器会自动添加，但会消耗资源，浏览器还会加错分号

  3. JS会自动忽略多个空格个换行

  4. JS中使用var声明变量

  5. 标识符**：** 在JS中由我们自主命名的变量名、函数名、属性名 （这点感觉和其它语言没什么区别。。

    • 标识符中可以包含字母、数字、_、$

    • 标识符不能以数字开头

    • 标识符不能是ES中的关键字或保留字

    • 标识符一般采用驼峰命名法

      首字符小写，每个单词开头字母大写，其余字母小写 如 var helloWorld = 1

    • JS底层保存标识符用Unicode编码（utf-8的内容都可以作为标识符）

  6. JS只有六种数据类型：

    typeof 检查变量类型

    1）String 字符串

    ​ -单引号内可以放双引号

    ​ -或者使用 ’\‘ 做转义字符

    2）Number 数值

    ​ -JS中所有整数都是Number类型

    ​ -最大数值1.797e+308

    3）Boolean 布尔值

    4）Null 空值

    ​ -表示空对象

    5）Undefined 未定义

    6）Object 对象

  7. 类型转换

  8. 进制表示

    16进制： 0x开头

    8进制：0开头

    2进制：0b开头

  9. 运算符(操作符)

    运算符都会返回结果

    任何值和NaN做运算，结果都是NaN

    算术运算符： + - * /%

    字符串"+":（拼串操作）

    ​ -写长字符串的时候，可以借助"+"对代码进行换行，更加美观

    ​ -任何值和字符串相加，都会被转为字符串

    ​ -利用a = a + “” 将任意类型转为string（隐式的类型转换

​ 除了"+"以外，参与运算的元素都会被转换为Number

​ Number的隐式转换:

​ -为一个值 -0 *1 /1

​ -或者使用一元运算符 + / -

  1. 逻辑运算符

    ! : 非 布尔值取反 对非布尔值，会将其先转换为布尔值，再取反。

    Boolean的隐式转换：

    -两次非运算

    && : 与

    || : 或

    && || 非布尔值的情况:

    && 如果第一个为false，则返回第一个值；第一个为true，则返回第二个值

    || 如果第一个为true，则直接返回第一个值; 第一个为false, 则返回第二个值

  2. 关系运算符

    非数值情况:

    ​ 对于非数值进行比较时，会将其转换为数字然后比较

    ​ 如果符号两侧的值都是字符串，则会比较字符的Unicode编码，一位一位进行比较

    ==: 相等

    ​ 会自动进行类型转换，将其转会为相同的类型，然后再比较

    ​ undefine衍生自null，所以二者判断相等时，会返回true

    可以通过isNaN(), 判断一个值是否是NaN

​ !=: 不相等：

​ 不相等也会对变量进行自动的类型转换

​ ===: 全等

​ 不会做类型转换，如果两个值类型不同，直接返回false

​ !==: 不全等
